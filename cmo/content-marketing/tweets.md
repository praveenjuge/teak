### Thread 3: How We Built Teak's AI Pipeline

1/7
Technical thread: How Teak processes content with AI in real-time.

2/7
When you save a link to Teak, a lot happens behind the scenes:

1. Fetch the page
2. Extract metadata (title, description, images)
3. Detect content type
4. Run AI classification
5. Generate tags and summary
6. Create visual thumbnail

3/7
We use Convex workflows to orchestrate this pipeline. Each step runs independently, in parallel where possible.

4/7
For AI, we use Groq with multiple models:
- Fast models for classification
- Smart models for summarization
- Vision models for images
- Audio models for transcription

5/7
The trick: prompt caching. We cache shared prompts and see ~50% cost savings on token usage.

6/7
We also show optimistic UI updates. The card appears instantly while AI processes in the background.

7/7
Result: You save something, and it's fully processed with tags, summary, and metadata in seconds.

Open source at github.com/praveenjuge/teak-convex-nextjs
