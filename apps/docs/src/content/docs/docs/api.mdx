Teak provides a public API at `https://teakvault.com/api/v1`.

> The API moved from `api.teakvault.com` to `teakvault.com/api`. Existing
> `api.teakvault.com` URLs and API keys keep working, but new integrations
> should use the simpler `teakvault.com/api` address.

## Base URLs

- Production: `https://teakvault.com/api/v1`
- Local development: `http://docs.teak.localhost:1355/api/v1`
- OpenAPI: `https://teakvault.com/api/openapi.json`
- Legacy production: `https://api.teakvault.com/v1`

## MCP Endpoint

Teak also exposes a remote MCP server:

- Production: `https://teakvault.com/mcp`
- Local development: `http://docs.teak.localhost:1355/mcp`
- Legacy production: `https://api.teakvault.com/mcp`

See the dedicated [MCP docs](./mcp) for tool contracts and JSON-RPC examples.
API discovery always returns the canonical production MCP URL.

## Authentication

All API requests use a bearer token in the `Authorization` header. Two token types are accepted:

- **OAuth access token** — obtained by signing in with your browser (used by browser-login clients such as Raycast and MCP). Access tokens expire after 1 hour and are refreshed automatically by the client.
- **Teak API key** — a long-lived `teakapi_` key you generate in Settings.

```bash
Authorization: Bearer teakapi_secret_live_a1b2c3d4_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

Generate and manage API keys in Teak Settings: `https://app.teakvault.com/settings`.

## Headers

- `X-Request-Id` is returned on every API response.
- `Idempotency-Key` is supported on `POST /v1/cards` and `POST /v1/cards/bulk`.
- Rate-limited responses may include `RateLimit-Limit`, `RateLimit-Remaining`, `RateLimit-Reset`, and `Retry-After`.

## Endpoints

### Service Health

```http
GET /healthz
```

```json
{
  "status": "ok",
  "service": "teak-api",
  "version": "v1"
}
```

### API Info

```http
GET /v1
```

Response includes the current v1 route list plus MCP connection details.

### Canonical Card Listing

```http
GET /v1/cards?q=design&limit=25&cursor=opaque&include=content,metadata
```

Supported query parameters:

- `q`
- `limit`
- `cursor`
- `sort=newest|oldest`
- `type`
- `tag`
- `favorited=true|false`
- `createdAfter`
- `createdBefore`
- `include=content,metadata,processing`

The `include` parameter is a comma-separated list of field groups to add to each item. Omitting it returns the base fields only.

| `include` value | Extra fields added                                                |
| --------------- | ----------------------------------------------------------------- |
| `content`       | `content`, `notes`, `aiSummary`                                   |
| `metadata`      | `fileName`, `fileSize`, `mimeType`, `fileUrl`, `linkPreviewImageUrl`, `screenshotUrl`, `thumbnailUrl` |
| `processing`    | `processingStatus`, `metadataStatus`, `aiTranscript`              |

Response:

```json
{
  "items": [
    {
      "id": "card_1",
      "appUrl": "https://app.teakvault.com/?card=card_1",
      "createdAt": 1,
      "isFavorited": true,
      "metadataTitle": "Example",
      "tags": ["design"],
      "type": "link",
      "updatedAt": 2,
      "url": "https://example.com"
    }
  ],
  "pageInfo": {
    "hasMore": false,
    "nextCursor": null
  }
}
```

### Create Card

```http
POST /v1/cards
Content-Type: application/json
Idempotency-Key: save-link-123
```

Supported request body fields:

- `content` — text or URL to save (required if `url` is not set)
- `url` — URL to save (required if `content` is not set). Must be an `http` or `https` URL.
- `fileKey` — key returned by `POST /v1/uploads` for file cards
- `cardType` — optional with `fileKey`; Teak infers `image`, `video`, `audio`, or `document` from the file name and media type. Existing clients may keep sending it, and mismatches are rejected.
- `fileName`, `mimeType`, `fileSize` — file metadata for uploaded cards
- `notes` — optional freeform notes attached to the card (string or null)
- `tags` — optional array of tag strings
- `source` — optional string identifying the client or integration (e.g. `"raycast_quick_save"`)

```json
{
  "content": "https://teakvault.com",
  "tags": ["design"]
}
```

```json
{
  "status": "created",
  "cardId": "card_1",
  "appUrl": "https://app.teakvault.com/?card=card_1"
}
```

### Upload File

File cards use a direct upload flow so large bytes do not pass through the API.

```http
POST /v1/uploads
Content-Type: application/json
```

```json
{
  "fileName": "image.png",
  "mimeType": "image/png",
  "fileSize": 12345
}
```

The response includes a presigned `uploadUrl` and `fileKey`. Upload the bytes with `PUT`, then create the card with `POST /v1/cards`.

```json
{
  "method": "PUT",
  "uploadUrl": "https://...",
  "fileKey": "users/user_1/file/...",
  "maxFileSize": 104857600,
  "expiresIn": 600
}
```

```http
PUT <uploadUrl>
Content-Type: image/png
```

```http
POST /v1/cards
Content-Type: application/json
Idempotency-Key: upload-image-123
```

```json
{
  "fileKey": "users/user_1/file/...",
  "fileName": "image.png",
  "mimeType": "image/png",
  "fileSize": 12345,
  "tags": ["reference"]
}
```

The maximum file size is 100 MB. Supported uploads include JSON and design-token files; TSX, JSX, Vue, Svelte, HTML, CSS, Markdown, and MDX source; GIF, WebM, MP4, WebP, AVIF, SVG, and HEIC media; PDF, DOCX, PPTX, TXT, and RTF documents; ZIP archives; and Figma `.fig` files. Generic `application/octet-stream` uploads are accepted when the file name identifies a supported format.

File responses can include `fileKind`, `fileExtension`, `fileLanguage`, and compact `filePreview` facts. Extracted source or document text is used only while generating AI tags and summaries and is not returned or stored as raw text.

### Bulk Card Operations

```http
POST /v1/cards/bulk
Content-Type: application/json
Idempotency-Key: bulk-ops-001
```

```json
{
  "operation": "favorite",
  "items": [{ "cardId": "card_1", "isFavorited": true }]
}
```

Supported operations:

- `create`
- `update`
- `favorite`
- `delete`

A single request accepts at most 100 items. Requests with an empty `items`
array, or more than 100 items, are rejected with `400 INVALID_INPUT`. Split
larger batches across multiple requests.

Response:

```json
{
  "operation": "favorite",
  "results": [{ "index": 0, "status": "success", "cardId": "card_1" }],
  "summary": { "total": 1, "succeeded": 1, "failed": 0 }
}
```

### Legacy Search Endpoints

These are not deprecated and will keep working for existing consumers, but Teak does not add new features to them. New integrations should use `GET /v1/cards` instead, which supports the same filters plus cursor pagination and the `include` parameter.

```http
GET /v1/cards/search?q=design&limit=50
GET /v1/cards/favorites?q=design&limit=50
```

They accept the same query parameters as the canonical listing (`q`, `limit`, `type`, `tag`, `sort`, `favorited`, `createdAfter`, `createdBefore`) and return a flat response:

```json
{
  "items": [],
  "total": 0
}
```

### Get Card

```http
GET /v1/cards/:cardId
```

Returns the full card object.

### Update Card

```http
PATCH /v1/cards/:cardId
Content-Type: application/json
```

Allowed fields:

- `content`
- `url` — must be an `http` or `https` URL
- `notes`
- `tags`

Example:

```json
{
  "notes": "Updated note",
  "tags": ["reference"]
}
```

### Delete Card

```http
DELETE /v1/cards/:cardId
```

Response: `204 No Content`

### Favorite State

```http
PATCH /v1/cards/:cardId/favorite
Content-Type: application/json
```

```json
{
  "isFavorited": true
}
```

### Tags

```http
GET /v1/tags
```

```json

}
```

## Errors

Errors preserve the stable `{ code, error }` shape. Some responses may also include `requestId`, `details`, or `retryAt`.

Upload validation returns `TYPE_MISMATCH` when a filename extension conflicts with its declared MIME type and `FILE_TOO_LARGE` when the file exceeds the advertised upload limit.

```json
{
  "code": "INVALID_INPUT",
  "error": "Body must include `content` or `url`"
}
```

### Idempotency Key Errors

When using `Idempotency-Key` on `POST /v1/cards` or `POST /v1/cards/bulk`, two `409 Conflict` scenarios may occur:

| Situation                                              | `code`     | `error`                                                     |
| ------------------------------------------------------ | ---------- | ----------------------------------------------------------- |
| A request with the same key is still in flight         | `CONFLICT` | `Idempotency-Key is already being processed`                |
| The key was already used with a different request body | `CONFLICT` | `Idempotency-Key was already used with a different request` |

If the key was previously used with an identical request body, the original response is replayed without re-processing the operation.
